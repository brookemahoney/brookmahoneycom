Random Things, Catching Up

Things. Stuff. Blog. React. Drupal. Gatsby. Tome. React-Router. Remix. Epic Stack. Drupal Stack. Fly.io. Typescript. Rusty dev chops.

Took a couple of years off. Now, I'm rusty; getting back into things. Been watching lots of recent Drupal presentations lately, mostly from DrupalCon and from GovCon. Also learned about the Epic Stack, which is what Kent Dodds has been working on. I've spent a lot of hours relearning Typescript and React.

Why have I been gone? Life got in the way, as an old manager of mine used to say. The last big company I worked for was a very successful startup, then Covid hit, and it was a wild ride after that. It became so wild that it wasn't a good fit for me anymore. I was with them for about 5 years, so it was sad to leave.

This first blog post here on my brand new super fancy site is a bit of a dump. It's ok. I have a headache today. It just reminded me of that.

I'm really looking forward to working full time again. I just need to find a good company that could use someone who likes working with Drupal and/or React. In the meantime, I'm going to spend my days working on learning what best practices have changed.

This site is hosted on Github pages, which is kinda cool. They host the code too. So, I just am paying for a domain and email and hosting is done for free. I just have to keep the source code for the site public, which is fine. Open source it all, ok. I was using Vite with a simple React Router implementation. For now, I'm using vanilla HTML and CSS.

So, I suppose I'll migrate this site over to Drupal with Gatsby. I'll develop the Drupal site and back it up locally with Apple's Time Machine. I'll use Gatsby to export my Drupal site to a static site. This method should work good for my home site with its simplistic blog that I want to keep this year and it will be a good way to showcase some of my Drupal and React skills.

Aye, aye, AI

Oh, AI. Forgot to talk about that. OMG, everyone is talking about it and apparently a lot people really like using it for web development. I've heard it described as a junior developer with no memory. To me, it's just not that helpful. I like to read manuals and understand every bit of code I write. I'm not one of those programmers who likes to move fast and make lots of mistakes. I like careful planning.

I think that's enough writing for now. It's been nice talking about this stuff with you. Maybe I'll try it again later.
