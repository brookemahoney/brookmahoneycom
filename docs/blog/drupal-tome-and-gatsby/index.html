Drupal Tome and Gatsby

Let's do Drupal Gatsby

Deep breath. OK, I really want this site to be a Drupal Gatsby site. Let's plan it out. I'll read through the docs on the Drupal Gatsby project pages first, to see if there's a recipe available. Then, using Orbstack and DDEV I'll spin up a new drupal instance. I think it makes sense to have the root folder live in a subfolder of the main repo... like how about 'drupal'.

Let's do Tome

Found this: "You may use any static site generator in conjunction with Tome by reading exporting JSON directly. See tome-gatsby for an example of a Gatsby integration."

"If you want to use Drupal to generate HTML, Tome ships with a static site generator that uses Drupal's internal request handling and concurrency to ensure that pages are exported as quickly as possible." https://tome.fyi/docs/technical/static/

Aside: I asked them if they meant "reading exported JSON" instead: https://github.com/drupal-tome/drupal-tome.github.io/issues/35

So, If I use Tome, I can use their generator, or Gatsby's, or others. I think it makes sense to Condiser using Tome, especially since the University of Maryland started using it to host 1900 pages, according to a presentation given a few months ago.
