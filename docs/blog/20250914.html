Old Distributions remind me of Drupal CMS

Years ago, I used to tell people that Drupal distributions should be used as examples for building sites instead of using them as a starting code base and extending them. Drupal distributions were great but sometimes they caused headaches when you wanted to override their functions. This is how I feel about Drupal CMS, so far. I will use it as an example and I'll build my own site with Drupal Core as a starting point. I'll be taking a look at each of the core and contributed modules that ship with Drupal CMS, taking note of which ones I'll need for the blog.

Modules I'll start with

Pathauto and Redirect should do. I'll disable all the modules that don't make sense for a static site.
